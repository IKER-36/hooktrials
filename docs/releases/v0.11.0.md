# HookTrials v0.11.0 — competition-ready evidence

This release turns the one-click Guided Demo into a complete, inspectable product workspace.

## Highlights

- Separate Trial, Observe, Protect recovery and Protect dead-letter routes preserve each part of the
  reliability story instead of overwriting one demo endpoint.
- Webhook Hub shows three clearly labelled synthetic connections after the run.
- The recovery path sends a GitHub-shaped event with a valid `X-Hub-Signature-256`, enforces its
  inbound contract, survives two destination failures and succeeds on the third delivery.
- Control Center automatically selects the recovered route. It reaches 100/100 Production Readiness
  on the managed HTTPS deployment and exposes the complete Reliability Replay.
- Evidence sharing now proves the protected recovery, while Monitoring, public status, Operations,
  alert audit and dead-letter recovery remain populated.
- Browser ingestion uses origin-restricted CORS without credentials, enabling realistic signed demo
  requests while keeping Cloud origins explicit.
- Landing CTAs preserve the selected language and take new users directly to Guided Demo or Webhook
  Hub after authentication.

## Verification

The release passed formatting, ESLint, strict TypeScript, 133 automated tests, all production builds
and a browser-driven eight-step run on the full Docker Compose stack. The run produced three Webhook
Hub routes, five monitor types, seven audited demo alerts, one protected recovery, one actionable
dead letter, a public status page and a redacted evidence link.
